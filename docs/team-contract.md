## 5. Standar Commit Message
Menggunakan bahasa Inggris dan format singkat (imperative), contoh:
- Add login feature
- Fix bug on register page
- Update README

---

## 6. Workflow Git
- Branch utama:
  - `main`: rilis stabil
  - `dev`: integrasi harian
- Branch fitur:
  - format: `feature/nama-fitur`
- Alur kerja:
  1. Buat branch dari `dev`
  2. Commit perubahan dengan pesan jelas
  3. Push branch dan buat PR ke `dev`
  4. PR direview minimal 1 anggota
  5. Setelah disetujui, merge ke `dev`
  6. Sinkronisasi `main` dilakukan saat milestone selesai

---

## 7. Mekanisme Eskalasi
- Jika ada anggota tidak aktif → diingatkan di grup
- Jika masih tidak aktif → lapor ke ketua
- Jika terjadi masalah besar → konsultasi ke asisten dosen
